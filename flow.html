Start: Project Setup

Environment & Database Selection

Hardware: Ryzen 5 5600, 32GB RAM

Databases: MongoDB (NoSQL) & CockroachDB (NewSQL)

Dataset: Amazon Fashion Dataset

Clients: PyMongo (MongoDB) & Psycopg2 (CockroachDB)

Repeat for Each Database (MongoDB & CockroachDB)

Benchmark 1: Data Manipulation

Measure performance of INSERT, UPDATE, DELETE.

Data Size: 10k to 100k rows, in 10k increments.

Benchmark 2: Database Concurrency

Measure response time as simultaneous queries increase.

Concurrency Level: 2 to 5 simultaneous queries.

Benchmark 3: Query Optimization (Indexing)

Measure impact of an index on UPDATE performance.

Data Size: 10k to 100k rows, in 10k increments.

Benchmark 4: Resource Consumption

Measure CPU & Memory usage during bulk operations.

Data Size: 10k to 100k rows, in 10k increments.

Benchmark 5: Access Control Complexity

Measure p95 latency impact of roles under concurrent load.

Concurrency Level: 10, 50, 100, 500, 1000 users.

Benchmark 6: Data Integrity Constraints

Measure performance cost of enforcing data rules on INSERT.

Data Size: 10k to 100k rows, in 10k increments.

Analysis & Visualization

Record all execution times and resource metrics.

Generate line charts and tables to compare results.

Discuss findings and draw conclusions.
